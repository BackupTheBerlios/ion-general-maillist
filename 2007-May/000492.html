Source-based packages that already download the official tarball,
might also simply want to use the 

	http://iki.fi/tuomov/dl/ion-3-latest.tar.gz

(and .asc for signature) redirection. Obviously the package may
break when I release a new version, but that should give the
package maintainer a bit more incentive in updating it...

--

Also forgot to mention that the dead man switch DB would only
have to be downloaded by the package maintenance tools' update/
sync process. An alternative approach currently apparently taken 
by Debian is to check the package's site for the version on every
install, which is quite ugly.
