Hi Tuomo, thanks for the patch. Unfortunately I still experience problems with
frames getting stuck at 0 width or height after doing a 'Mod1-K V' or 'Mod1-K H'
in a neighboring frame. Fortunately I can reproduce it reliably:

1) Start with a 'full' tiled workspace containing a single frame.
2) Mod1-S to do a vertical split
3) In the top frame repeated 'Mod1-K V' toggle the bottom frame from 0 height
and then back to half the screen height, as expected.
4) Again in the top frame do a 'Mod1-K S' to do a horizontal split.
5) In one of the top frames do a 'Mod1-K V' to collapse the bottom frame. Then
'Mod1-TAB' to the other top frame and 'Mod1-K V'... the bottom frame
unexpectedly stays at 0!

(The same thing happens using 'Mod1-K H' in horizontal-split frames.) The key to
reproducing the problem is to switch between maximized frames while the
minimized frame is at 0 height/width, it then becomes impossible to resize the
minimized frame from the keyboard.

Let me know if you need a better description (or if I'm experiencing some
user-error/misconception).
