On 2007-05-13, Tuomo Valkonen <tuomov at iki.fi> wrote:
> I think it (and other modules I don't care for) should somehow indicate
> that it will void upstream support. This could be done with names like
> 'mod_xinerama-for-ion3', 'ion3-support-void-mod_xinerama', etc., but
> an unignorable notification might also suffice.

The above package names are, of course a bit cumbersome. One possibility
pointed out to me, would be to name unofficial modules according to the
scheme 'mod_(supportvoid|unsupported|unofficial)_xinerama'. Then the 
distributions would not have to give their packages peculiar names.
I don't want to force module authors to use this scheme [1], but I 
do recommend using it.

  [1]: They're not the problem. The problem is distributions 
       effectively hiding this and other information (version,
       patches, etc.) that the user would gain by using the 
       original sources.
