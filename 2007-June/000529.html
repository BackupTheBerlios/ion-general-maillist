According to the thread found here:
http://www.mail-archive.com/ion-general@lists.berlios.de/msg02067.html
the problem can be fixed by turning off dynamic linking. I tried to do this
by changing /sw/fink/10.4/unstable/main/finkinfo/x11-wm/ion.info as follows:


[...]
Revision: 2
[...]
ConfigureParams: --with-lua-prefix=%p --mandir=%p/share/man
--infodir=%p/share/\
info --libexecdir=%p/sbin --with-libtool=%p --with-disable-shared
[...]

however, even with the --diable-shared flag (found in the configure file of
the current ion-2 source package),

fink rebuild ion
fink reinstall ion

did not produce a working ion install.
