On 2007-03-21, Jeremy Hankins <nowan at nowan.org> wrote:
> The slightly tricky bit would be to account for the different sorts of
> options passed in the binding (_, _sub, etc.).  Simplest would be to make
> different ways of calling the lua function different commands, where
> necessary.

You'd end up devising something like the context menu definitions
and generation code. You need to define commands for the contexts
they are available in, and that's what the context menus do.
The Mod1+M query_menu is a sort of command query already.

It's not a lot of work to add all the usual commands in the
context and other menus, and some extra options to prune out
the entries from the menus that would not normally be needed
in them. However, maintaining separate documentation is extra
work.
